> 最后更新：2026-06-23

## 文心大模型 API（erniebot SDK）

**选型原因：** 赛题要求使用飞桨文心大模型 API

**模型分工：**

| 模型 | 定位 | 任务类型 |
|------|------|----------|
| ernie-lite | 轻量快速 | 意图识别、摘要、分类、实体提取、翻译、通用对话 |
| ernie-3.5 | 强力复杂 | 代码分析、诊断、邮件生成、文献综述、推理、任务规划、报告 |

**降级链：** ernie-3.5 → ernie-lite → 本地 fallback

**路由机制：** config.py MODEL_ROUTING 表，13种 task_type 自动选择模型

**环境变量：**
- `ERNIEBOT_ACCESS_TOKEN` — 主 token（ernie-lite + ernie-3.5 共用）
- `ERNIEBOT_STRONG_TOKEN` — ernie-3.5 专用 token（可选，为空则共用主 token）

**设计原则：** 每次调用超时 30s，超时自动切换；记录每步调用结果（成功/失败/耗时/token）

## 持久化选型

**决策：SQLite + JSONL 双写**

| 方案 | 优点 | 缺点 | 结论 |
|------|------|------|------|
| /tmp 目录（原方案） | 简单 | 重启丢失，无法查询 | ❌ 已废弃 |
| SQLite（当前） | 重启持久，支持查询，无外部依赖 | 并发写入有限 | ✅ **选用** |
| PostgreSQL | 功能完整 | 需要外部服务，黑客松环境不适合 | ❌ |

**实现：**
- `data/traces/` — JSONL 格式，每次状态跳转追加写入（可追溯）
- `data/checkpoints/` — SQLite 数据库，任务状态快照（支持恢复）
- 双写策略：每次 `_write_trace_record` 同时写 JSONL 和 SQLite

## 安全体系选型

**决策：多层防御（纵深防御）**

| 层级 | 组件 | 职责 |
|------|------|------|
| 工具层 | 工具白名单 | 按状态/阶段限制可用工具 |
| 操作层 | 危险操作检测 | 正则匹配 13 种危险命令模式（critical/high/medium） |
| 确认层 | 四值确认机制 | allow / always / deny / exit（学自 OpenVibeCoding） |
| 预算层 | Token 预算 | per-state/per-phase 预算，超限强制退出 |
| 隐私层 | PrivacyGuard | PII 脱敏（手机号/身份证/银行卡/邮箱/密码） |
| 通信层 | InputScanner | Agent 间通信安全扫描（注入攻击/PII/危险指令） |
| 进展层 | ProgressDetector | 防 Agent 循环推诿，连续重复输出自动熔断 |

## 上下文管理选型

**决策：滑动窗口 + 摘要压缩 + 响应缓存**

| 方案 | 优点 | 缺点 | 结论 |
|------|------|------|------|
| 全量历史 | 信息完整 | Token 消耗线性增长 | ❌ |
| 无历史 | Token 最少 | 丧失上下文 | ❌ |
| 滑动窗口（当前） | 保留最近 K 轮 + 早期压缩为摘要 | 压缩有损 | ✅ **选用** |

**实现：**
- `ContextWindow`：保留最近 10 轮完整对话，早期对话压缩为摘要
- `ContextAwareLLM`：自动注入上下文、管理窗口、响应缓存
- 响应缓存：低温任务（temperature<0.3）相同输入自动命中，TTL 5分钟，200条上限

## 执行后端选型

**决策：Brain/Hands 分离架构**

| 后端 | 场景 | 实现 |
|------|------|------|
| 本地进程 | 默认执行 | 直接 subprocess 调用 |
| DockerHands | 多租户隔离 | `docker run --rm` 容器执行 |
| MockHands | 测试/CI | 确定性输入→确定性输出 |

**设计原则：** 换执行后端不影响编排逻辑，编排层可独立测试

## MCP 协议选型

**决策：纯 Python JSON-RPC over stdio**

| 方案 | 优点 | 缺点 | 结论 |
|------|------|------|------|
| HTTP REST | 生态好 | 需要额外服务 | ❌ |
| gRPC | 高性能 | Python 依赖重 | ❌ |
| JSON-RPC stdio（当前） | 零依赖，纯 Python | 文档较少 | ✅ **选用** |

**优势：** 零侵入扩展，新工具只需实现 MCP Server 接口
