## 一、项目概述

### 1.1 应用场景

在 deepin 25 操作系统上，基于 OpenClaw 多智能体框架和文心大模型 API，构建一个**多智能体协作系统**，实现复杂任务的自动化拆解与协同执行。

典型应用场景：用户提出一个高层需求（如"帮我分析这个项目的代码结构并生成文档"），系统自动拆解为多个子任务，由不同角色的 Agent 并行/串行执行，最终整合结果返回给用户。

### 1.2 项目目标

- 在 deepin 25 上完成 OpenClaw 环境部署
- 基于文心大模型 API（erniebot）实现多 Agent 协作
- 完成至少 2 个不同场景的演示（文献综述 + 代码分析）
- 输出完整的可复现 Demo

## 二、技术架构

### 2.1 核心组件

| 组件 | 技术选型 | 说明 |
|------|---------|------|
| 操作系统 | deepin 25 | 任务目标环境 |
| Agent 框架 | OpenClaw | 多智能体协作框架 |
| 大模型 | 文心大模型（erniebot） | 通过 ERNIE-4.5 或 ernie-lite 调用 |
| 编程语言 | Python 3.10+ | 主力开发语言 |
| 运行环境 | WSL2 / 原生 Linux | 部署方式 |

### 2.2 Agent 角色设计

```
Lead Agent（主智能体）
├── 负责任务拆解和结果整合
├── 直接与用户交互
└── 调用文心大模型进行推理

Researcher Agent（研究智能体）
├── 负责信息检索和文献分析
├── 输出结构化的研究发现
└── 支持多轮问答

Coder Agent（编码智能体）
├── 负责代码分析和文档生成
├── 执行 Python/Shell 命令
└── 输出分析报告
```

### 2.3 协作流程

```
用户输入 → Lead Agent 拆解任务
                        ↓
          ┌─────────────┼─────────────┐
          ↓             ↓             ↓
    Researcher   Coder   角色N    （并行执行）
          ↓             ↓             ↓
          └─────────────┼─────────────┘
                        ↓
              Lead Agent 整合结果 → 用户输出
```

## 三、场景演示

### 场景一：项目代码分析 + 文档生成

**输入：** 一个代码仓库路径  
**流程：**
1. Lead Agent 接收任务，拆分为"读取代码结构"+"分析核心逻辑"+"生成文档"
2. Researcher Agent 遍历目录，提取文件清单和依赖关系
3. Coder Agent 分析关键文件的逻辑，生成摘要
4. Lead Agent 整合输出 Markdown 格式的代码文档

**预期输出：** 自动生成的项目文档，包含目录结构、模块说明、关键函数列表

---

### 场景二：文献综述助手

**输入：** 多个 PDF/文本文件路径 + 研究问题  
**流程：**
1. Lead Agent 接收问题，拆解为"提取各文献要点"+"对比分析"+"生成综述"
2. Researcher Agent 并行读取各文献，提取关键信息
3. Lead Agent 综合分析，生成综述报告

**预期输出：** 结构化的文献综述文档，含各文献要点对比表

## 六、技术难点与解决方案

### 难点 1：OpenClaw 在 deepin 上的兼容性
- **解决：** 使用 Python 虚拟环境隔离依赖，通过 WSL2 规避部分库兼容问题
- **备选：** 如遇底层库问题，考虑 Docker 容器化部署

### 难点 2：多 Agent 之间的通信协调
- **解决：** 基于 OpenClaw 内置的 Agent 通信机制实现任务分发
- **备选：** 自行实现简单的消息队列协调

### 难点 3：文心 API 调用成本控制
- **解决：** 合理设计 Prompt，控制 Token 消耗
- **备选：** 优先使用 ernie-lite 模型
