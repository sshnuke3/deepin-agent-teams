## 一、项目概述

**deepin-agent-teams** 是一个运行在 deepin 25 桌面环境上的**多智能体协作系统**，基于文心大模型 API 实现复杂任务的自动拆解与协同执行。

### 核心亮点

| 特性 | 说明 |
|------|------|
| 🧠 智能路由 | ScenarioClassifier「三道筛子」自动识别场景，中英文双语支持，无需手动切换 |
| ✍️ 灵活输入 | 支持中/英文输入，英文确认回复主动建议（yes/ok/sure等） |
| 👁️ 主动感知 | 剪贴板/窗口/系统三级感知，用户没下指令也能自主行动 |
| 🔺 三角架构 | 状态机引擎 + 独立 Verifier + Worker 池，职责分离 |
| 🛡️ 四层安全 | 输入扫描 → 工具白名单 → 确认机制 → 隐私保护 |
| 🤖 纯文心 | ernie-lite + ernie-3.5 双模型路由，无第三方模型 |

## 三、核心模块说明

### 3.1 ScenarioClassifier — 智能场景路由

**三道筛子**判断用户输入是否适合 Agent 化：

| 筛子 | 判断标准 | 示例 |
|------|---------|------|
| 输入模糊性 | 非结构化 → 需要理解 | "帮我写封邮件" ✅ / "打开浏览器" ❌ |
| 跨系统协调 | 需要多个工具联动 | "查资料写报告" ✅ / "翻译这段话" ❌ |
| 多步推理 | 4+ 步骤才能完成 | "分析代码并优化" ✅ / "今天天气" ❌ |

**三都不过** → 直接 LLM 对话（轻量，不进状态机）
**任一过了** → 进入 Agent 状态机（完整流程）

**集成方式**：`ChatWindow` 初始化时接收 `classifier` 参数，用户输入自动路由：

```python
classifier = ScenarioClassifier()
chat_window = ChatWindow(scenarios, classifier=classifier)
# 用户输入 → classifier.classify() → 自动选择场景 → 执行
```

### 3.2 感知桥接 — 主动感知能力

| 感知类型 | 轮询间隔 | 触发条件 | 响应方式 |
|---------|---------|---------|---------|
| 剪贴板 | 2 秒 | 检测到英文内容 | 自动翻译 → 剪贴板 |
| 窗口切换 | 3 秒 | 打开代码/邮件/文档 | 主动推荐服务 |
| 系统异常 | 10 秒 | 服务崩溃/资源告警 | 主动诊断告警 |

**关键改进**：`get_window_classification(title, app_class)` 现在同时接收窗口标题和应用类名，分类更精确。

### 3.3 三角架构 — 状态机 + Verifier + Worker

**状态机**（7 种状态，代码写死跳转条件）：
```
PENDING → CLAIMED → PLANNING → RUNNING → VERIFIED → COMPLETED
                                  ↓ FAIL
                                RETRY（≤3次）
                                  ↓ 超次
                                FAILED
```

**Verifier**（独立质检，11 项检查清单）：
- 基础检查：交付物存在、功能正确、trace 完整、无错误
- 安全检查：工具合规、Token 预算、危险操作确认
- 规划检查：计划完整性、依赖关系自洽
- 上下文检查：Token 溢出、摘要质量

**模型路由**（13 种 task_type）：

| task_type | 模型 | 场景 |
|-----------|------|------|
| intent / summary / classify / entity / translate / general | ernie-lite | 轻量快速 |
| email / diagnosis / code / literature / reasoning / task_plan / report | ernie-3.5 | 复杂推理 |

## 四、演示流程（4 个场景，约 4 分钟）

### 场景一：启动 + 智能路由对话（0:00 - 0:45）

**目的**：展示 GUI 界面和 ScenarioClassifier 自动路由

**操作步骤**：

1. 终端启动：
   ```bash
   cd ~/Desktop/deepin-agent-teams
   source venv/bin/activate
   python main.py --gui
   ```
2. 等待 2 秒，桌面右下角出现 🦞 悬浮球（带呼吸动画）
3. 点击悬浮球 → 对话窗口展开
4. 输入：
   ```
   帮我分析一下这个项目的代码结构
   ```
5. **观察**：对话窗口自动识别为 `code` 场景，显示 🔍 代码分析
6. 等待 Agent 返回分析结果

**字幕**：`ScenarioClassifier 自动路由：输入 → 三道筛子 → code 场景 → Agent 状态机`

---

### 场景二：剪贴板感知 → 自动翻译（0:45 - 1:45）⭐ 重点

**目的**：展示「用户没有下指令，Agent 自主执行」

**操作步骤**：

1. 对话窗口最小化（点 − 按钮）
2. 打开浏览器或记事本，选中一段**英文技术文章**，Ctrl+C 复制
3. **不做任何操作，等待 3-5 秒**
4. 观察悬浮球：
   - 悬浮球变**蓝色**，图标变成 📋
   - 呼吸灯动画
5. **悬浮球变绿色**，图标变成 ✅
6. 对话窗口自动弹出，显示：
   > ✅ 已翻译并复制到剪贴板：
   > [翻译结果]
7. 打开编辑器，Ctrl+V 粘贴，确认剪贴板已是中文

**字幕**：`感知 → 决策 → 自主执行：confidence 85% + risk=low → 自动翻译`

---

### 场景三：窗口切换感知 → 意图预测（1:45 - 2:45）⭐ 重点

**目的**：展示「系统知道用户在做什么，主动提供帮助」

**操作步骤**：

1. 对话窗口最小化
2. 打开文件管理器，浏览到包含 `.py` 文件的目录
3. 双击打开一个 Python 文件（用编辑器）
4. **不做任何操作，等待 3-5 秒**
5. 观察悬浮球：
   - 悬浮球变**绿色**，图标变成 🪟
6. 对话窗口自动弹出，显示：
   > 🐍 检测到您正在查看 Python 代码，需要代码分析还是优化建议？
7. 回复：`分析一下这段代码`
8. 等待 Agent 返回分析结果

**字幕**：`感知链路：窗口切换(3s) → get_window_classification(title, app_class) → 意图预测`

---

### 场景四：系统异常感知 → 主动诊断（2:45 - 3:30）⭐ 重点

**目的**：展示「系统级主动感知，用户没有报错，系统自己发现异常」

**操作步骤**：

1. 对话窗口最小化
2. 打开终端，执行：
   ```bash
   sudo systemctl stop cups
   ```
3. **不做任何操作，等待 5-10 秒**（系统监控轮询间隔 10 秒）
4. 观察悬浮球：
   - 悬浮球变**红色**，图标变成 ⚠️
5. 对话窗口自动弹出，显示：
   > ⚠️ 检测到系统服务异常（cups），需要自动诊断吗？
6. **用英文回复**：`yes` 或 `diagnose it`
7. 等待 Agent 返回诊断报告和修复建议

**字幕**：`感知链路：系统服务监控(10s) → 异常检测 → 主动告警 → 英文确认 → 自动诊断`

---

**字幕**：`ScenarioClassifier 中英文双语支持：英文输入 → 自动路由 → Agent 执行`

## 七、录制注意事项

1. **每个「等待 3-5 秒」是关键** — 不能跳过，这是「主动感知」的核心证据
2. **悬浮球状态变化要拍到** — 变色、图标变化、呼吸灯是视觉证据
3. **对话窗口必须是自动弹出的** — 不能是用户手动打开的
4. **场景之间暂停 2-3 秒** — 让评委看清状态变化
5. **不需要麦克风** — 只录屏幕，字幕说明即可
6. **系统感知场景等待时间较长（10秒）** — 可以在等待时展示其他操作

## 八、关键改进（v2 → v3）

| 改进项 | v2 | v3 |
|--------|----|----|
| 场景路由 | 手动选择 tab | ScenarioClassifier 自动路由 |
| 语言支持 | 仅中文 | 中英文双语（关键词+筛子+复杂度） |
| 主动建议确认 | 仅中文确认 | 中英文确认（yes/ok/sure/好的等） |
| 窗口分类 | 仅 title | title + app_class 双参数 |
| 对话窗口 | 固定场景 | 单输入框 + 智能意图路由 |
| 系统监控 | 服务状态格式错误 | 修复字典格式，正确触发告警 |
| 剪贴板 | 仅读取 | 支持读取+写入（set_clipboard_text） |
| 架构描述 | 混合层 | 三角架构（状态机 + Verifier + Worker）|
