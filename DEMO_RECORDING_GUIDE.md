## 一、录制前准备

### 1.1 环境初始化

```bash
cd ~/Desktop/deepin-agent-teams
source venv/bin/activate
python --version  # 确认 Python 3.11+
```

### 1.2 检查 ffmpeg

```bash
ffmpeg -version | head -1
```

### 1.3 清理桌面

- 关闭不必要的窗口
- 保持桌面简洁（只留终端和关键窗口）
- 准备好演示素材：
  - 一段英文技术文章（浏览器打开或记事本里备好）
  - 一个 Python 文件（用编辑器打开）
  - 终端（用于触发系统异常）

### 1.4 开始录制

```bash
ffmpeg -f x11grab -r 30 -s 1280x800 -i :0.0 -c:v libx264 -preset fast ~/Desktop/deepin_demo.mp4
```

## 二、演示流程（4 个场景，约 3 分 45 秒）

### 场景一：悬浮球激活 + 被动响应（0:00 - 0:30）

**目的**：展示 GUI 界面和基本交互能力

**操作步骤**：

1. 终端输入：
   ```bash
   python main.py --gui
   ```
2. 等待 2 秒，桌面右下角出现 🦞 悬浮球（带呼吸动画）
3. 用鼠标点击悬浮球 → 对话窗口展开
4. 在对话窗口输入：
   ```
   帮我查一下系统状态
   ```
5. 等待 Agent 返回系统诊断报告
6. 截图确认：对话窗口显示诊断结果

**字幕**：`基本交互：悬浮球 → 对话窗口 → 自然语言指令`

---

### 场景二：剪贴板感知 → 主动推荐（0:30 - 1:30）⭐ 重点

**目的**：展示「用户没有下指令，系统主动发现」

**操作步骤**：

1. 将对话窗口最小化（点 − 按钮）
2. 打开浏览器或记事本，选中一段**英文技术文章**，Ctrl+C 复制
3. **不做任何操作，等待 3-5 秒**
4. 观察悬浮球：
   - 悬浮球变**蓝色**
   - 图标变成 📋
   - 开始呼吸灯动画（大小交替）
5. **同时**，对话窗口自动弹出，显示：
   > 🔍 检测到您复制了一段英文内容，需要翻译还是总结？
6. 在对话窗口回复：
   ```
   翻译成中文
   ```
7. 等待 Agent 返回翻译结果

**字幕**：
```
感知链路：剪贴板监听(2s) → 内容识别(英文/代码/URL) → 意图推断 → 主动推荐
```

---

### 场景三：窗口切换感知 → 意图预测（1:30 - 2:30）⭐ 重点

**目的**：展示「系统知道用户在做什么，并预测需要什么帮助」

**操作步骤**：

1. 对话窗口最小化
2. 打开文件管理器，浏览到一个包含 `.py` 文件的目录
3. 双击打开一个 Python 文件（用编辑器）
4. **不做任何操作，等待 3-5 秒**
5. 观察悬浮球：
   - 悬浮球变**绿色**
   - 图标变成 🪟
   - 呼吸灯动画
6. 对话窗口自动弹出，显示：
   > 🐍 检测到您正在查看 Python 代码，需要代码分析还是优化建议？
7. 回复：
   ```
   分析一下这段代码
   ```
8. 等待 Agent 返回代码分析结果

**字幕**：
```
感知链路：窗口切换事件(3s) → 应用分类(代码/邮件/文档) → 上下文构建 → 意图预测
```

---

### 场景四：系统异常感知 → 主动诊断（2:30 - 3:30）⭐ 重点

**目的**：展示「系统级主动感知，用户没有报错，系统自己发现异常」

**操作步骤**：

1. 对话窗口最小化
2. 打开终端，执行：
   ```bash
   sudo systemctl stop cups
   ```
   （模拟打印机服务故障）
3. **不做任何操作，等待 5-10 秒**（系统监控轮询间隔是 10 秒）
4. 观察悬浮球：
   - 悬浮球变**红色**
   - 图标变成 ⚠️
   - 呼吸灯动画
5. 对话窗口自动弹出，显示：
   > ⚠️ 检测到系统服务异常（cups），需要自动诊断吗？
6. 回复：
   ```
   诊断一下
   ```
7. 等待 Agent 返回诊断报告和修复建议

**字幕**：
```
感知链路：系统服务监控(10s) → 异常检测 → 主动告警 → 自动诊断
```

---

### 收尾（3:30 - 3:45）

在对话窗口或终端展示架构总结：

```
感知层：剪贴板监听 / 窗口管理 / 系统监控
    ↓
桥接层：PerceptionBridge（轮询 + 信号分发）
    ↓
交互层：悬浮球变色 / 对话窗口主动弹出 / 托盘通知
    ↓
执行层：状态机引擎 / Worker 池 / 独立 Verifier
```

## 三、完整操作序列（复制粘贴用）

```bash
# === 场景一：启动 GUI ===
```

## 四、录制参数

- **帧率**：30fps

- **编码**：H.264

## 五、注意事项

1. **每个「等待 3-5 秒」是关键** — 不能跳过，这是「主动感知」的核心证据
2. **悬浮球状态变化要拍到** — 变色、图标变化、呼吸灯是视觉证据
3. **对话窗口必须是自动弹出的** — 不能是用户手动打开的
4. **场景之间暂停 2-3 秒** — 让评委看清状态变化
5. **不需要麦克风** — 只录屏幕，字幕说明即可
6. **系统感知场景等待时间较长（10秒）** — 可以在等待时展示其他操作，但不要错过悬浮球变色的瞬间
